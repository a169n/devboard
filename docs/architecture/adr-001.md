# ADR-001: Stateless JWT Auth for MVP

## Status
Accepted

## Context
DevBoard needs authenticated multi-user ownership without enterprise complexity.

## Decision
Use JWT bearer tokens for stateless API authentication and bcrypt password hashing.

## Consequences
- Simple deployment and scaling (no server sessions).
- Token revocation is coarse-grained (expiration based) for MVP.
- Frontend stores token in localStorage; acceptable for MVP with documented trade-off.
